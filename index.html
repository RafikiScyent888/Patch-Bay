7 Attributes

1
Network ID (NID) — Default Gateway
The first IP in the subnet. It's where the block of addresses starts. Not available to assign to a device.
2
Broadcast ID (BID)
The last IP in the subnet. It's where the block of addresses stops. Not available to assign to a device.
3
First Host ID
The first IP you can actually assign — NID + 1.
4
Last Host ID
The last IP you can actually assign — BID − 1.
5
Next Network ID (NNID)
Where the next subnet starts — BID + 1.
6
Number of IPs
Total addresses in the block is 2ⁿ, where n = 32 − CIDR. Subtract 2 (the NID and BID aren't assignable) to get the number of IPs you can actually hand out.
7
CIDR / Subnet Size
The /number after the IP. It sets how big the subnet is — how many IP addresses fall inside it.

Finding a Target IP — the Block-Size Method

1

Starting at 0, keep adding the group size (the number of IPs per subnet) to get each Network ID in the sequence: 0, 0 + size, 0 + size + size, and so on.

2

Stop as soon as you pass the target IP. The number you landed on just before it is that subnet's Network ID — the next number in the sequence is the Next Network ID, where the following subnet begins.

3

An octet can't go higher than 255 (there are 256 values once you count 0). If adding the group size would go past 255, that step rolls over into the next octet instead.

VLSM

Variable Length Subnet Masking — carving one network into differently-sized blocks so every group gets exactly the room it needs, instead of handing everyone the same size subnet.

1

List every group's required host count, then sort them largest to smallest. The biggest group always gets allocated first.

2

For each group, find the smallest CIDR that still covers its host count — use the cheat sheet in Subnet Calc above, or check that 2ⁿ − 2 (where n = 32 − CIDR) is at least as big as what's needed.

3

Place the biggest group starting at the network's first address. Every group after that has to start at an address that's a clean multiple of its own block size — if the previous block doesn't end on one, round up to the next multiple before starting.

4

Every block still loses its first and last address to the Network ID and Broadcast ID — same as any subnet, subtract 2 from the block size to get that group's usable hosts.

Every IPv4 address is 32 bits split into four 8-bit octets. A /24 fills the first three octets — 24 bits — with network bits and leaves one whole octet, 8 bits, for hosts:

1928 bits · network
1688 bits · network
18 bits · network
08 bits · hosts

8 host bits gives 2⁸ = 256 addresses; subtract the Network ID and Broadcast ID and you're left with 254 usable hosts — that's the plain /24 baseline before VLSM starts carving out smaller, differently-sized blocks.

IPv6

128-bit addresses written as eight groups of four hex digits — IPv4 exhaustion is why this exists, and every one of the four certs now expects you to at least read and shorten one on sight.

Format & Compression Rules

1
128 bits, 8 groups
Written as eight groups of four hex digits separated by colons — 2001:0db8:85a3:0000:0000:8a2e:0370:7334.
2
Drop leading zeros — per group
Each group can lose its own leading zeros: 0db8 → db8, 0000 → 0. Never drop trailing zeros.
3
Collapse one run of zero groups with ::
One consecutive run of two or more all-zero groups can be replaced with a double colon — but only one run per address, and only if it's the longest run present.
4
A lone zero group doesn't get ::
A single isolated 0000 group just becomes 0 — :: is reserved for runs of two or more, so it always unambiguously expands back out.

Binary / Hex

Subnet masks, MAC addresses, IPv6 groups, memory addresses — it's all binary and hex underneath. Fluency here is what makes the rest of this page fast instead of a lookup-table exercise.

One Byte, Three Ways

DecimalBinaryHex
0000000000x00
15000011110x0f
128100000000x80
255111111110xff
1

Binary → decimal: add up the place values of every 1 bit. Left to right, an 8-bit byte's place values are 128, 64, 32, 16, 8, 4, 2, 1.

2

Decimal → hex: a byte is exactly two hex digits, one per nibble (4 bits). Split the byte in half and convert each half 0–15 into a single hex digit.

3

Binary ↔ hex is the fastest conversion of the three — each hex digit maps to exactly 4 bits, no decimal math required at all.
